The ReAct Method

Reasoning and Acting. A powerful technique for complex problem-solving that makes AI thinking transparent and verifiable.

Reason

Think through the problem step by step

+
Act

Take actions based on reasoning

=
ReAct

Transparent, verifiable solutions

What is ReAct?

ReAct (Reasoning + Acting) is a prompting paradigm where the AI alternates between thinking through a problem and taking actions. This creates a transparent chain of thought that you can follow and verify.

Explicit Reasoning

The AI shows its work, explaining why it's making each decision.

Interleaved Actions

Reasoning leads to concrete actions, which produce new information for more reasoning.

Verifiable Process

You can check each step, catch errors early, and understand the full logic.

When to Use ReAct

Use ReAct for complex problems where the solution isn't obvious, when you need to verify reasoning, or when the task requires multiple steps with decisions along the way.

The ReAct Loop

1

Thought

Analyze the current state and decide what to do next

2

Action

Take a specific action based on the reasoning

3

Observation

Note the result of the action

This loop continues until the problem is solved. Each iteration builds on the previous one, creating a clear trail of logic.

How to Prompt for ReAct

To get ReAct-style responses, structure your prompt to encourage explicit reasoning:

ReAct Prompt Template

Task: [Your problem or question]

Instructions: Solve this step by step. For each step:

  1. Thought: Explain your reasoning about what to do next
  2. Action: Describe the action you're taking
  3. Observation: Note the result and what it tells you

Continue until you reach the solution.

ReAct Prompt Example

"I need to debug why our API is returning 500 errors intermittently. Walk me through your diagnostic process step by step. For each step, share your reasoning (Thought), what you would check or do (Action), and what the result tells us (Observation). Continue until you identify the likely cause."

ReAct in Action

Here's how a ReAct-style response looks for a debugging task:

Thought 1

Intermittent 500 errors suggest the issue isn't with the code logic itself (which would fail consistently), but likely related to resources, connections, or external dependencies. I should first identify if there's a pattern to when errors occur.

Action 1

Check the error logs for timestamps and look for patterns. Are errors clustered at certain times? Do they correlate with traffic spikes?

Observation 1

Errors spike between 9-10 AM and 2-3 PM, which corresponds to peak usage times. The errors include "connection timeout" messages.

Thought 2

Connection timeouts during peak usage suggests either database connection pool exhaustion or the external service we depend on is rate-limiting us. I should check which connections are timing out.

Action 2

Examine the stack traces in the logs to identify which service call is timing out - is it the database or an external API?

Observation 2

The timeouts are occurring on calls to our payment processor API, not the database.

Conclusion

The payment processor is likely rate-limiting our requests during peak times. We should implement request queuing, caching of results where appropriate, and consider upgrading our API tier with the payment processor.

Best Use Cases for ReAct

Debugging & Troubleshooting

Work through issues systematically, testing hypotheses and narrowing down causes.

Research & Analysis

Explore a topic by gathering information, synthesizing findings, and drawing conclusions.

Decision Making

Evaluate options, consider trade-offs, and arrive at a reasoned recommendation.

Complex Calculations

Break down multi-step problems, showing work at each stage.

Planning & Strategy

Consider constraints, evaluate approaches, and build a step-by-step plan.

Code Review

Systematically analyze code, explain reasoning, and provide justified feedback.
