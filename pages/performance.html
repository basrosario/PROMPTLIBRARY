Core Web Vitals

Google's official metrics for user experience quality

LCP: Largest Contentful Paint

Measures how quickly the largest content element becomes visible. Google defines "Good" as under 2.5 seconds. Praxis achieves sub-second LCP through efficient CSS, deferred JavaScript, and no render-blocking resources.

Replaced FID in March 2024 as the responsiveness metric. Measures how quickly the page responds to all user interactions throughout the session. Target is under 200ms. Vanilla JS ensures immediate feedback.

Optimization Techniques

How we achieve and maintain perfect performance

CSS Preloading

Stylesheet loaded with rel="preload" hint tells browsers to fetch CSS immediately. Critical styles arrive before page parsing begins.

Deferred JavaScript

All scripts use the defer attribute. HTML parsing continues uninterrupted. Scripts execute after DOM is ready without blocking rendering.

System Fonts

No external font loading. System font stack is already on the user's device. Zero font-related delay, instant text rendering.

Single Origin

Everything from one domain. No CDN lookups, no CORS preflight, no third-party failures. One connection, maximum efficiency.

Inline SVG Icons

Icons are inline SVG, not external fonts or images. They render instantly, scale perfectly, and add negligible weight.

Clean DOM

500-800 DOM nodes versus 1,500+ typical sites. Semantic HTML without framework wrapper divs means faster parsing and rendering.

What Zero Dependencies Means

Every decision in Praxis architecture was made with performance in mind. Here is what our zero-dependency approach delivers.

No Framework Runtime

0KB of framework code. Every byte is actual functionality, not library overhead.

No External Fonts

System fonts load instantly. No FOUT, no layout shifts from font swapping.

No Analytics Trackers

No Google Analytics, no pixels. Faster loads plus complete privacy.

No Build Pipeline

What we write is what ships. No webpack, no babel, no transpilation bugs.

Network Efficiency

Optimized for real-world connection conditions

Gzip Compression

All text resources compressed by the server. Actual transfer sizes are 60-70% smaller than raw file sizes.

Cache Headers

Static assets have long cache lifetimes. Returning visitors load from local cache, making subsequent visits instant.

No CORS Requests

Same-origin means no preflight requests, no cross-origin handshakes, no blocked requests from misconfiguration.

Minimal DNS

Only one domain to resolve. Sites with CDNs, fonts, analytics require multiple DNS lookups, each adding latency.

HTTP/2 Ready

Single domain maximizes HTTP/2 connection reuse. Multiple domains fragment connections, hurting performance.

Mobile Optimized

Light pages load fast on 3G/4G. No hero videos, no massive images. Works well on metered connections.
