Security as a Practice

Security isn't something you do once and forget. According to NIST's Cybersecurity Framework, effective security requires continuous monitoring, assessment, and improvement[1]. Praxis was designed with security as a core principle, not a bolt-on addition.

Defense-in-Depth Architecture

Multiple security layers as recommended by NIST SP 800-53

The defense-in-depth model, as defined by NIST Special Publication 800-53, recommends implementing multiple overlapping security controls so that if one fails, others remain[2].

Defense in Depth

Multiple layers of security controls ensure that if one layer fails, others remain. We don't rely on a single security measure but implement overlapping protections at every level.

Multi-Layer Protection

Principle of Least Privilege

Our Content Security Policy allows only what's absolutely necessary. No inline scripts, no external resources, no unnecessary permissions. If it's not needed, it's blocked.

Content Security Policy Analysis

A+ rated CSP verified against Mozilla Observatory standards

Content Security Policy (CSP) is a browser security mechanism that helps prevent cross-site scripting (XSS), clickjacking, and other code injection attacks. According to NIST's Web Services Security Guidelines, CSP is one of the most effective mitigations against XSS attacks[4].

CSP Directives Explained

1

default-src 'none'

This is the most restrictive default policy possible. Every resource type must be explicitly allowed. This follows the principle of deny-by-default recommended by CISA[5].

Effect

Prevents any unexpected content from loading, including scripts, styles, images, fonts, and connections unless explicitly permitted.

2

script-src 'self'

Only scripts from our own domain are allowed. No inline scripts, no eval(), no external JavaScript files from CDNs.

XSS Prevention

According to CMU CERT/CC security research, strict CSP that blocks inline scripts can prevent the majority of XSS attacks[6].

3

style-src 'self'

Only stylesheets from our domain are permitted. No inline styles means no CSS injection attacks possible.

Implementation

All 11,000+ lines of CSS are in a single external stylesheet (styles.css). Zero style attributes in HTML.

4

connect-src 'self'

Network requests only to our own domain. No external API calls, no tracking beacons, no data exfiltration possible.

Privacy Benefit

User data cannot be sent to third parties because the browser blocks all external connections.

5

form-action 'none'

Forms cannot submit data anywhere. This prevents form hijacking attacks where malicious actors redirect form submissions to attacker-controlled servers.

Praxis Note

Since all tools run client-side in JavaScript, no form submissions are needed.

6

base-uri 'none'

The base URL cannot be changed by injected content. This prevents base tag injection attacks that could redirect all relative URLs to attacker-controlled servers.

Why This Matters

According to CISA and NIST, Cross-Site Scripting (XSS) remains a critical web application security risk[7]. Most websites allow inline scripts and styles for convenience. We chose the harder path: refactoring all code to work with strict CSP. The result is a site that's practically immune to XSS attacks.

Complete Privacy Protection

Privacy by Design, as formalized by Dr. Ann Cavoukian and endorsed by international privacy authorities, establishes that privacy should be embedded into technology from the start[8]. Praxis implements all seven foundational principles.

No Analytics

No Google Analytics, no tracking pixels, no session recording. We don't know who you are, where you're from, or what you're doing.

No Cookies

Common Vulnerabilities We Prevent

How our architecture eliminates top web security risks

Cross-Site Scripting (XSS)

Status: Blocked - Strict CSP prevents any injected scripts from executing, whether inline or from external sources.

Supply Chain Attacks

Status: Eliminated - Zero external dependencies means no npm vulnerabilities, no CDN compromises, no third-party risks.

Data Exfiltration

Status: Impossible - CSP blocks all external connections. No data can be sent to attacker-controlled servers.

Clickjacking

Status: Prevented - Frame-ancestors directive prevents embedding in malicious iframes on other sites.

Form Hijacking

Base Tag Injection
