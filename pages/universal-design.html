Understanding Universal Design

Design for all, not retrofit for some

The Core Principle

Design for the Edges

Universal Design, as defined by NC State's Center for Universal Design, means creating products and environments usable by all people, to the greatest extent possible, without adaptation or specialized design[3].

When we design for the edges, everyone benefits. Curb cuts help wheelchair users—but also parents with strollers, travelers with luggage, and delivery workers with carts.

  • Proactive design

    Built in from the start, not retrofitted

  • One version for all

    No separate "accessible version" needed

  • Better for everyone

    Accessible design improves the experience for all users

The Curb Cut Effect

When Berkeley, California added curb cuts for wheelchair users in the 1970s, they discovered that 9 out of 10 people using them didn't use wheelchairs. Parents with strollers, delivery workers, travelers with luggage, and cyclists all benefited. This became known as the "curb cut effect"—design for accessibility helps everyone.

The Three UDL Principles

CAST.org's framework for inclusive education[4]

Multiple Means of Engagement

The "Why" of Learning — Different learners are motivated in different ways. Praxis offers interactive tools for hands-on learners, comprehensive guides for readers, and quizzes for those who learn through challenge.

Motivation

Multiple Means of Representation

The "What" of Learning — Information should be presented in multiple formats. Praxis uses text, visual cards, interactive examples, color coding, and progressive disclosure for different minds.

Perception

Multiple Means of Action

The "How" of Learning — Learners differ in how they demonstrate knowledge. Analyze prompts, build step-by-step, match methods, take quizzes, or simply read—all valid paths.

Expression

UDL is built on three core principles based on neuroscience research about how humans learn. These address the "why," "what," and "how" of learning. Praxis implements all three throughout the platform.

The 7 Principles of Universal Design

Originated at NC State's Center for Universal Design, 1997[3]

1

Equitable Use

Same content and tools available to all users. No separate "accessible version"—the main site IS the accessible version.

Praxis Implementation

All pages, tools, and features work identically for all users regardless of ability.

2

Flexibility in Use

Multiple pathways to the same knowledge. Choose your preferred learning method.

Praxis Implementation

Tools, guides, quiz, glossary—learn the way that works best for you.

3

Simple and Intuitive

Clear navigation, consistent patterns, plain language. No prior knowledge required.

Praxis Implementation

Consistent UI patterns, 550+ glossary terms, progressive complexity from basics to advanced.

4

Perceptible Information

Information presented in multiple formats. High contrast, scalable text, icons with labels.

Praxis Implementation

Text + visual cards + icons + color coding. 4.5:1+ contrast ratios throughout.

5

Tolerance for Error

Interactive tools provide feedback, not judgment. Wrong answers lead to learning.

Praxis Implementation

Quiz explains correct answers. Prompt Analyzer shows improvements, not failures.

6

Low Physical Effort

No time pressure. No rapid interactions required. Fast-loading, lightweight pages.

Praxis Implementation

100% Lighthouse performance score. No timed interactions. Single CSS + JS files.

7

Size and Space for Use

Generous tap targets on mobile. Readable text sizes. Responsive layouts.

Praxis Implementation

44x44px minimum touch targets. Text scaling up to 3x. Full responsive design.

WCAG 2.1 Compliance

The POUR principles: Perceivable, Operable, Understandable, Robust[2]

Perceivable

Information must be presentable in ways users can perceive. Text alternatives for images (1.1), adaptable content (1.3), distinguishable with 4.5:1+ contrast (1.4).

Level AA Met

Operable

UI components must be operable. Keyboard accessible (2.1), no time limits (2.2), no seizure triggers (2.3), navigable with skip links (2.4).

Level AA Met

Understandable

Content and operation must be understandable. Readable with lang attribute (3.1), predictable behavior (3.2), input assistance with error identification (3.3).

Level AA Met

Robust

Content must be robust enough for assistive tech. Valid HTML parsing (4.1.1), name/role/value exposed properly (4.1.2), status messages announced (4.1.3).

Level AA Met
Section 508 Compliance

WCAG 2.1 Level AA is incorporated by reference into Section 508 of the U.S. Rehabilitation Act[5]. Meeting WCAG AA means Praxis complies with U.S. federal accessibility requirements for electronic and information technology.

Color Contrast Analysis

Meeting and exceeding WCAG requirements

12.6:1 Primary Text Exceeds AA by 2.8x
8.3:1 Secondary Text Nearly 2x required
5.9:1 Link Text Exceeds AA by 31%
+15% High Contrast Mode Additional boost available

WCAG AA requires 4.5:1 contrast for normal text and 3:1 for large text. Praxis exceeds these requirements throughout, with primary text at 12.6:1 (dark gray #111827 on white). Color is never the sole means of conveying information—icons, text, and patterns reinforce meaning for color-blind users.
