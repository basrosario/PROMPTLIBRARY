The Process

AI-Assisted Development Workflow

How each feature moves from idea to implementation

1. Human Vision

Human describes the goal: what to build, why it matters, and how it should work. Creative direction and strategic decisions come from human insight.

Example Request

"Add a glossary page with 500+ AI terms, alphabetical navigation, and search functionality."

2. AI Exploration

Claude reads existing code to understand patterns, finds similar implementations, and gathers context. This ensures new code matches established conventions.

  • Analyzes existing page structures
  • Reviews CSS component patterns
  • Checks JavaScript conventions

3. AI Implementation

Claude generates code, creates files, and makes edits. Progress is visible in real-time, and humans can intervene or redirect at any point.

  • Writes semantic HTML structure
  • Adds CSS following design system
  • Implements JavaScript functionality

4. Human Review

Human reviews changes, tests functionality, checks visual appearance. Adjustments requested as needed. Nothing merges without human approval.

Review Focus

Does it work correctly? Does it look right? Does it match the vision?

5. Commit with Attribution

Changes committed with descriptive messages and "Co-Authored-By: Claude" attribution. Full transparency about AI involvement in every commit.

Real Example

AI Collaboration in Action

How a typical feature request becomes implemented code

Feature: Adding a New Learn Page
Human Request

Create a new learn page for the COSTAR framework. It should follow the same structure as the CRISP page—hero section, formula display, element breakdown with examples, and practical exercises. Match the existing visual style and ensure CSP compliance.

AI Analysis

Reading crisp.html to understand structure... Found: hero section, framework-formula component, element-timeline for breakdown, comparison-panels for examples. CSS classes use BEM naming. Will replicate pattern with COSTAR content.

AI Implementation

Created costar.html with matching structure. Added C-O-S-T-A-R formula display. Built element-timeline with Context, Objective, Style, Tone, Audience, Response sections. All inline styles avoided. Navigation updated across 25 pages.

The Same Principles Apply

Notice how this workflow mirrors good prompting: clear task (create a new page), relevant context (follow CRISP structure), specific constraints (CSP compliance, match visual style). The same communication skills that make prompts effective also make AI-assisted development successful.

Human Essential

What AI Cannot Replace

AI is a powerful tool, but understanding its limits is as important as knowing its capabilities. These areas remain uniquely human:

Original Ideas

AI implements; humans imagine. The vision for Praxis—making AI accessible to everyone—is entirely human.

Aesthetic Judgment

AI generates code for designs but doesn't know what "looks good." Color, spacing, visual hierarchy require human taste.

User Empathy

AI doesn't feel the frustration of a confused beginner. Human empathy shapes user experience decisions.

Business Decisions

What to prioritize, when to ship, what trade-offs to accept—these strategic calls remain human.

AI Makes Mistakes

AI-generated code can have bugs, introduce security issues, or miss requirements. Research from MIT CSAIL confirms that AI code generation requires human oversight for quality assurance[2]. Every change is reviewed before acceptance—trust but verify.

Quality Assurance

Maintaining Professional Standards

How AI-assisted development achieves production-quality results

CLAUDE.md Instructions

A project-level instruction file defines coding standards, file organization, and conventions. AI follows these rules consistently, producing code that matches existing style.

Insights

Lessons from AI-Assisted Building

What we learned building Praxis with Claude Code

Key Takeaways
Clear Context Matters

AI works best with clear constraints and established patterns. The CLAUDE.md file and HANDOFF.md system dramatically improved consistency and reduced errors across sessions.

Review Everything

AI makes confident mistakes. Always verify output against intent. Catch problems early when they're easy to fix. Trust the process, not the output blindly.

Humans Stay Essential

AI accelerates implementation but doesn't replace judgment. The hardest problems—what to build, why, for whom—remain deeply human questions.

Iteration Works

Perfect first drafts are rare. Ask for adjustments, refinements, alternatives. The collaboration improves through multiple passes, just like human teamwork.

The Meta-Lesson

Building Praxis taught us exactly what we now teach: effective AI collaboration requires clear communication, structured methodology, and iterative refinement. NSF research on human-AI teaming confirms that explicit communication patterns improve collaboration outcomes[3].
